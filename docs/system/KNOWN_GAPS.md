# Known Gaps / TODOs (Docs)

This pack aims to be “AI-audit-ready”, but a few areas are intentionally summarized rather than fully spec’d.

## 1) Outbox payload schemas (per-type)
`OUTBOX_EVENTS.md` documents the *intent* and typical payload keys, but some event types have payloads that vary by source.

If you need exact schemas:
- Search queue sites for `type: "<event>"` and inspect the `.values({ payload: ... })` shape.
- Inspect the handler logic in `StonegateOS/apps/api/src/lib/outbox-processor.ts` for required keys.

## 2) Full request/response schemas for every endpoint
`API_ROUTE_INDEX_*.md` lists endpoints + methods + handlers, but we do not yet include:
- Request body schemas (zod) for every route
- Response payload schemas for every route
- Example curl requests

## 3) “Business logic invariants”
We do not yet have a single formal spec for invariants like:
- “When a lead is created, pipeline stage must be updated”
- “When appointment is canceled, calendar conflict rules must ignore it”

These are partly encoded in code + `CRITICAL_FLOWS.md`.

## 4) Regenerating the generated docs
The following files are generated by local scripts/commands:
- `StonegateOS/docs/system/API_ROUTE_INDEX_API.md`
- `StonegateOS/docs/system/API_ROUTE_INDEX_SITE.md`
- `StonegateOS/docs/system/OUTBOX_EVENT_INDEX.md`
- `StonegateOS/docs/system/DATA_DICTIONARY_COLUMNS.md`

If handlers/routes change, regenerate those before using this pack as “current truth”.
